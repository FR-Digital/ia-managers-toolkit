# Glossaire Complet IA pour Managers

> Tous les termes IA que vous pourriez rencontrer, classés par thème.

---

## A

### Accuracy (Précision globale)
Pourcentage de prédictions correctes sur l'ensemble des cas. Attention : peut être trompeuse si les classes sont déséquilibrées.

### Agent IA
Système IA capable d'agir de manière autonome pour atteindre un objectif, en prenant des décisions et en utilisant des outils.

### Algorithme
Suite d'instructions pour résoudre un problème. En IA, désigne souvent la méthode d'apprentissage utilisée.

### Annotation
Processus de labellisation des données d'entraînement par des humains. Exemple : marquer chaque email comme "spam" ou "pas spam".

### API (Application Programming Interface)
Interface permettant à des logiciels de communiquer entre eux. Utilisée pour appeler les services IA (ex : OpenAI API).

### Apprentissage Automatique
Voir Machine Learning.

### Apprentissage Non Supervisé
Le modèle apprend des patterns sans labels prédéfinis. Utilisé pour le clustering et la détection d'anomalies.

### Apprentissage par Renforcement
Apprentissage par essai-erreur avec système de récompenses. Utilisé pour l'optimisation et les jeux.

### Apprentissage Profond
Voir Deep Learning.

### Apprentissage Supervisé
Le modèle apprend à partir d'exemples étiquetés (input + output attendu).

### AUC-ROC (Area Under Curve)
Métrique qui mesure la capacité d'un modèle à discriminer entre classes. Score de 0.5 = hasard, 1 = parfait.

---

## B

### Backpropagation
Algorithme d'entraînement des réseaux de neurones, qui ajuste les poids en fonction des erreurs.

### Baseline
Point de référence pour comparer les performances. Peut être une solution simple (moyenne, règles) ou le système existant.

### Batch Size
Nombre d'exemples traités ensemble pendant l'entraînement.

### Biais (Bias)
Préjugés ou déséquilibres présents dans les données qui se retrouvent dans les prédictions de l'IA.

### Big Data
Très grands volumes de données nécessitant des outils spécialisés pour le stockage et le traitement.

---

## C

### Chatbot
Robot conversationnel qui simule une conversation humaine, basé sur des règles ou sur l'IA générative.

### Classification
Tâche consistant à attribuer une catégorie à une entrée. Ex : spam/pas spam, positif/négatif.

### Clustering
Regroupement automatique de données similaires sans catégories prédéfinies.

### CNN (Convolutional Neural Network)
Type de réseau de neurones spécialisé dans l'analyse d'images.

### Computer Vision
Domaine de l'IA qui permet aux ordinateurs de "voir" et interpréter les images/vidéos.

### Confiance (Score de)
Degré de certitude de l'IA dans sa prédiction, généralement exprimé en pourcentage.

### Context Window
Quantité de texte qu'un LLM peut traiter en une seule fois (mesuré en tokens).

---

## D

### Data Augmentation
Techniques pour augmenter artificiellement la taille du dataset (rotations d'images, synonymes, etc.).

### Data Drift
Évolution des données réelles par rapport aux données d'entraînement, causant une dégradation des performances.

### Data Engineering
Discipline de construction et maintenance des pipelines de données.

### Data Lake
Stockage centralisé de données brutes de différents formats.

### Data Science
Discipline combinant statistiques, informatique et expertise métier pour extraire des insights des données.

### Dataset (Jeu de données)
Ensemble de données utilisées pour entraîner ou tester un modèle.

### Deep Learning
Sous-ensemble du ML utilisant des réseaux de neurones à plusieurs couches pour apprendre des représentations complexes.

### DPIA (Data Protection Impact Assessment)
Analyse d'impact sur la protection des données, requise par le RGPD pour certains traitements.

---

## E

### Edge Computing
Traitement des données au plus près de leur source (sur l'appareil) plutôt que dans le cloud.

### Embedding
Représentation numérique d'un objet (mot, image) dans un espace vectoriel, capturant ses caractéristiques sémantiques.

### Entraînement (Training)
Phase où le modèle apprend à partir des données, en ajustant ses paramètres.

### Epoch
Un passage complet sur l'ensemble des données d'entraînement.

### Éthique de l'IA
Ensemble des principes moraux guidant le développement et l'utilisation de l'IA (équité, transparence, responsabilité).

### Explicabilité (Explainability)
Capacité à comprendre et expliquer comment un modèle prend ses décisions.

---

## F

### F1-Score
Moyenne harmonique de la précision et du recall, utile pour évaluation équilibrée.

### Feature (Caractéristique)
Variable d'entrée utilisée par le modèle pour faire ses prédictions.

### Feature Engineering
Processus de création et sélection des features pertinentes pour améliorer les performances.

### Few-Shot Learning
Apprentissage à partir de très peu d'exemples (quelques uns).

### Fine-tuning
Adaptation d'un modèle pré-entraîné à une tâche spécifique avec des données supplémentaires.

### Foundation Model
Modèle de grande taille pré-entraîné sur de vastes données, servant de base pour diverses applications.

---

## G

### GAN (Generative Adversarial Network)
Architecture où deux réseaux s'affrontent : un générateur et un discriminateur.

### Généralisation
Capacité du modèle à bien performer sur des données jamais vues pendant l'entraînement.

### GenAI (Generative AI)
IA capable de générer du nouveau contenu (texte, images, code, musique).

### GPU (Graphics Processing Unit)
Processeur spécialisé accélérant les calculs nécessaires au ML.

### Ground Truth
Les vraies réponses/labels contre lesquels on évalue les prédictions du modèle.

---

## H

### Hallucination
Quand l'IA génère des informations fausses mais présentées avec confiance comme vraies.

### Hyperparamètres
Paramètres de configuration du modèle définis avant l'entraînement (learning rate, epochs, etc.).

---

## I

### IA Faible (Narrow AI)
IA spécialisée dans une tâche spécifique. C'est ce qui existe aujourd'hui.

### IA Forte (General AI)
IA hypothétique avec intelligence comparable à l'humain. N'existe pas encore.

### Inférence
Utilisation du modèle entraîné pour faire des prédictions sur de nouvelles données.

### Intent Classification
Identification de l'intention derrière une requête utilisateur.

---

## K

### K-Means
Algorithme de clustering populaire qui regroupe les données en K groupes.

---

## L

### Label (Étiquette)
La réponse correcte associée à un exemple dans un dataset supervisé.

### Latence
Temps de réponse du système, du moment de la requête à la réception de la réponse.

### Learning Rate
Vitesse à laquelle le modèle ajuste ses paramètres pendant l'entraînement.

### LLM (Large Language Model)
Modèle de langage de grande taille entraîné sur des milliards de textes. Ex : GPT, Claude, Llama.

---

## M

### Machine Learning (ML)
Sous-domaine de l'IA où les systèmes apprennent à partir de données sans être explicitement programmés.

### MAPE (Mean Absolute Percentage Error)
Erreur moyenne en pourcentage, utile pour les prédictions numériques.

### MLOps
Pratiques pour automatiser et industrialiser le cycle de vie des modèles ML en production.

### Modèle
Le "cerveau" de l'IA après entraînement, contenant les patterns appris.

### Multi-modal
Modèle capable de traiter plusieurs types de données (texte + images + audio).

---

## N

### Named Entity Recognition (NER)
Extraction d'entités nommées dans un texte (personnes, lieux, organisations).

### Neural Network (Réseau de neurones)
Architecture inspirée du cerveau humain, composée de couches de neurones interconnectés.

### NLP (Natural Language Processing)
Traitement automatique du langage naturel (texte, parole).

### NLU (Natural Language Understanding)
Compréhension du sens et de l'intention dans le langage naturel.

---

## O

### OCR (Optical Character Recognition)
Reconnaissance optique de caractères : convertir images de texte en texte éditable.

### On-premise
Déploiement sur ses propres serveurs (vs cloud).

### One-shot Learning
Apprentissage à partir d'un seul exemple par classe.

### Overfitting (Sur-apprentissage)
Quand le modèle mémorise les données d'entraînement au lieu de généraliser.

---

## P

### Parameter (Paramètre)
Valeur interne du modèle apprise pendant l'entraînement.

### Pipeline
Chaîne de traitement automatisée des données, de la collecte à la prédiction.

### POC (Proof of Concept)
Projet pilote pour démontrer la faisabilité technique.

### Precision (Précision du positif)
Proportion de vrais positifs parmi tous les éléments prédits positifs.

### Pré-entraînement
Entraînement initial d'un modèle sur de grandes quantités de données générales.

### Prompt
Instruction ou question donnée à un modèle génératif.

### Prompt Engineering
Art de formuler des prompts efficaces pour obtenir les meilleures réponses.

---

## R

### RAG (Retrieval Augmented Generation)
Architecture combinant recherche dans une base de connaissances + génération par LLM.

### Random Forest
Algorithme d'ensemble combinant plusieurs arbres de décision.

### Recall (Rappel)
Proportion de vrais positifs détectés parmi tous les vrais positifs existants.

### Recommender System
Système de recommandation suggérant des produits/contenus basés sur les préférences.

### Regression
Tâche de prédiction d'une valeur numérique continue (ex : prix, température).

### RLHF (Reinforcement Learning from Human Feedback)
Méthode d'entraînement utilisant les retours humains pour améliorer le modèle.

### ROI (Return on Investment)
Retour sur investissement : (Gains - Coûts) / Coûts × 100.

---

## S

### Semantic Search
Recherche basée sur le sens plutôt que les mots-clés exacts.

### Sentiment Analysis
Détection du sentiment (positif/négatif/neutre) dans un texte.

### Supervised Learning
Voir Apprentissage Supervisé.

---

## T

### Temperature
Paramètre contrôlant la créativité/randomness des réponses d'un LLM.

### Test Set (Jeu de test)
Données réservées pour évaluer la performance finale du modèle.

### Time Series
Données séquentielles dans le temps (ex : cours de bourse, météo).

### Token
Unité de texte traitée par un LLM (mot ou partie de mot).

### Training Set (Jeu d'entraînement)
Données utilisées pour entraîner le modèle.

### Transfer Learning
Réutilisation d'un modèle pré-entraîné pour une nouvelle tâche similaire.

### Transformer
Architecture de réseau de neurones à la base des LLM modernes.

---

## U

### Underfitting (Sous-apprentissage)
Quand le modèle est trop simple pour capturer les patterns des données.

### Unsupervised Learning
Voir Apprentissage Non Supervisé.

---

## V

### Validation Set (Jeu de validation)
Données pour ajuster les hyperparamètres pendant le développement.

### Vector Database
Base de données optimisée pour stocker et rechercher des embeddings.

---

## W

### Weight (Poids)
Paramètre numérique dans un réseau de neurones ajusté pendant l'entraînement.

---

## X

### XAI (Explainable AI)
IA explicable : techniques pour rendre les décisions de l'IA compréhensibles.

### XGBoost
Algorithme populaire de gradient boosting, efficace pour les données structurées.

---

## Z

### Zero-shot Learning
Capacité à effectuer une tâche sans aucun exemple d'entraînement spécifique.
